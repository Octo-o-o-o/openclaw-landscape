# openclaw-feishu (2235 stars)

## Problem & Solution

Feishu (Lark) users face pain points when connecting to OpenClaw, including complex configuration, scattered documentation, and insufficient Chinese ecosystem adaptation. This project provides a beginner-friendly configuration guide, a common issue troubleshooting checklist, and comparison/selection recommendations between the Feishu official plugin and the OpenClaw built-in plugin, lowering the onboarding barrier for non-technical users.

## Key Features

- **Three plugin options compared** — Feishu official plugin (OAuth user identity), OpenClaw built-in plugin (bot identity), community early plugin (no longer updated), with a detailed selection matrix
- **Zero-to-hero configuration tutorial** — 4-step guided installation (Auth -> Agent -> Install -> Channel), covering multiple model providers including Anthropic/OpenAI/Gemini
- **Common issue troubleshooting** — API quota exhaustion root cause analysis, Lark Webhook intranet tunneling solutions, version check bug workarounds
- **Chinese ecosystem adaptation** — Companion project [awesome-openclaw-usecases-zh](https://github.com/AlexAnys/awesome-openclaw-usecases-zh) with 39 real-world use cases (including Feishu/Lark, DingTalk, WeCom, and Xiaohongshu scenarios)
- **Migration guide** — Complete steps for migrating from HTTP callback mode to WebSocket long connection
- **Advanced configuration reference** — Configuration examples for advanced features including Bitable (multi-dimensional tables), calendar scheduling, task management, and knowledge base
