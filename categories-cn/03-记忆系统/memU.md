# memU (12,811 stars)

## 问题与解决方案
解决了 24/7 长时运行 Agent 的两大核心问题：上下文窗口膨胀导致的 token 成本爆炸，以及被动响应模式无法主动理解用户意图。memU 通过文件系统式记忆架构和主动意图捕获，让 Agent 能够持续在线、低成本运行，并在用户未发出指令时就能预判需求并主动行动。

## 核心架构
- **文件系统式记忆** — 将记忆组织为分层目录结构（preferences/relationships/knowledge/context），支持导航、挂载、交叉引用和导出
- **主动记忆生命周期** — Main Agent 处理用户查询，memU Bot 并行监控输入输出、提取洞察、预测意图、执行主动任务
- **持续同步循环** — Agent ◄──► memU Bot ◄──► DB 三方持续同步，记忆实时更新并注入到 Agent 上下文
- **成本优化** — 通过缓存洞察和避免冗余 LLM 调用，将长时运行 token 成本降低至约 1/10

## 关键特性
- **主动意图捕获** — 持续理解用户目标、偏好和上下文，无需显式指令即可预判需求
- **主动任务执行** — 信息推荐（根据阅读历史推荐论文）、邮件管理（自动草拟回复和标记优先级）
- **memU Bot** — 企业级 OpenClaw 替代方案，一键安装（< 3 分钟），开源且可自托管
- **记忆即文件系统** — 像浏览目录一样导航记忆，像挂载磁盘一样加载知识源
- **可移植性** — 记忆可导出、备份和迁移，支持跨 Agent 共享
