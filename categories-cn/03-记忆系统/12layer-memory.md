# openclaw-memory-architecture (31 stars)

## 问题与解决方案

AI Agent 的记忆系统通常只依赖单一的向量搜索，导致精确查询（如"我女儿的生日"）和模糊查询（如"上周关于部署的讨论"）都使用同一套机制，效率低下。该项目提出 12 层记忆架构，结合结构化存储、语义搜索、知识图谱和认知模式，为不同类型的记忆召回选择最优策略。

## 关键特性

- **12 层分层架构** — 从无损上下文引擎（LCM）到元认知管道，每层针对特定记忆类型优化
- **统一搜索接口** — `memory_search` 工具并行查询四个后端（continuity、facts、files、lcm），一次调用覆盖所有记忆系统
- **知识图谱集成** — 3,000+ 事实的知识图谱，支持实体关系查询和别名解析
- **语义搜索优化** — 使用 llama.cpp + nomic 768d 嵌入，7ms GPU 查询延迟，支持多语言
- **激活衰减系统** — 记忆按访问频率和时间衰减，自动分层存储（热 / 温 / 冷）
- **LightRAG 领域知识库** — 集成 11 本书 + 139 篇论文的 GraphRAG，支持深度领域查询
